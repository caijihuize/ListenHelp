## ListenHelp6 听力辅助应用

ListenHelp6 是一款专为**听力障碍人士**和有听力提升需求的用户设计的 Android 听力辅助应用。  
应用基于 **AAudio 低延迟音频 API + C++ 原生处理** 实现实时声音采集、增强与播放，帮助用户在嘈杂环境中获得更清晰、可控的听觉体验。

---

## 功能概览

- **实时听力辅助**：
  - 端到端低延迟音频链路（AAudio 独占、低延迟模式）
  - 后台前台均可持续运行（前台服务 + 唤醒锁）
- **多设备支持**：
  - 输入：内置麦克风、有线耳机麦克风、USB、蓝牙 SCO 等
  - 输出：手机扬声器、有线耳机、USB、蓝牙 A2DP 等
  - 支持“跟随系统”或手动指定输入/输出设备
- **音量与放大控制**：
  - 独立的输入音量、输出音量滑块（0–100）
  - 支持最大 100 倍放大系数（适配重度听力补偿场景）
- **降噪开关**：
  - 简单“噪声门”型降噪，可抑制低幅度背景噪声
- **8 段均衡器**：
  - 频段覆盖 60Hz–20kHz（60 / 230 / 910 / 1.8k / 3.6k / 7.2k / 14k / 20kHz）
  - 支持预设：自定义、平坦、低音增强、高音增强、人声增强、低音抑制
  - 每段 -15dB ～ +15dB，UI 通过滑块与 dB 文本展示
- **实时波形可视化**：
  - 自定义 `WaveformView`，分别展示输入/输出波形
  - 便于用户感知当前环境音与处理后信号差异
- **设置自动保存**：
  - 使用 `SharedPreferences` 持久化音量、放大倍数、降噪开关、均衡器及预设
  - 应用重启后自动恢复上次配置

## 架构与技术栈

- **平台与语言**
  - Android 原生应用（最低 `minSdk 24`，目标 `targetSdk 34`）
  - Java（UI & 业务逻辑）+ C++（AAudio 实时音频处理，经 JNI 调用）
- **核心模块**
  - `MainActivity`
    - 负责 UI 控件（设备选择、音量/放大/降噪、均衡器、波形）
    - 与 `AAudioManager` 交互，控制开始/停止音频处理
    - 管理权限请求、电池优化白名单提示、设置保存/加载
  - `AAudioManager`
    - Java 层音频核心管理类
    - 维护输入/输出设备、音量、放大倍数、降噪与均衡器状态
    - 通过 `AAudioProcessorJNI` 调用 C++ 层音频处理器
  - `AudioProcessingService`
    - 前台 Service，负责：
      - 创建通知与通知渠道，保证应用长时间后台稳定运行
      - 持有 `WakeLock` 防止 CPU 休眠
      - 与 `MainActivity` 绑定，维持音频处理生命周期
  - `AAudioProcessor`（C++）
    - 使用 AAudio 打开输入/输出流，独占 + 低延迟模式
    - 在回调中依次执行：输入音量 → 放大 → 降噪 → 均衡器 → 输出音量 → 写出
    - 向上层回调输入/输出波形数据用于 UI 绘制

## 环境要求

- **运行设备**
  - Android 7.0+（API 24+）；推荐 Android 8.0+ 以完整支持 AAudio
  - 设备需具备麦克风，并允许应用访问
- **开发环境**
  - Android Studio（建议使用较新版本 Iguana 及以上）
  - 已安装 Android SDK 24–34 对应平台与构建工具
  - 已安装 NDK（用于构建 C++ 音频模块）
  - 使用项目自带 Gradle Wrapper 构建

## 构建与运行

### 使用 Android Studio

1. **克隆项目**
   ```bash
   git clone https://github.com/your-name/ListenHelp.git
   cd ListenHelp
   ```
2. 在 Android Studio 中选择 **“Open an Existing Project”**，打开仓库根目录。  
3. 等待 Gradle 同步与 NDK 配置完成（首次会稍慢）。  
4. 连接一台真实 Android 设备（推荐，而非模拟器），确保：
   - 已开启开发者选项和 USB 调试
   - 系统版本 ≥ Android 7.0
5. 点击 **Run ▶** 选择设备，安装并启动应用。

### 使用命令行（可选）

在项目根目录执行：
```bash
./gradlew assembleDebug
```
生成的 APK 位于 `app/build/outputs/apk/debug/`，可通过 `adb install` 安装。

## 使用指南

1. **首次启动**
   - 打开应用后，根据系统提示授予 **麦克风**（以及 Android 14+ 的前台麦克风服务）权限。
   - 如遇到电池优化提示，请按引导将应用加入白名单，以保证后台稳定运行。
2. **选择设备**
   - 在“设备选择”卡片中：
     - 选择输入设备（麦克风），如内置麦克风、蓝牙耳机等；
     - 选择输出设备（扬声器/耳机），或保持“跟随系统”。
   - 如插拔设备或蓝牙重新连接，可点击“刷新设备”。
3. **调整音频参数**
   - 在“音频设置”卡片中：
     - 调整输入音量、输出音量；
     - 按需调整“放大系数”，注意高倍数可能非常响亮；
     - 切换“降噪”开关以抑制弱背景噪音。
4. **均衡器设置**
   - 在“均衡器”卡片顶部下拉框中选择预设（如“人声增强”“低音增强”等）；
   - 选择“自定义”后，可以单独拖动 8 个频段滑块，针对不同频率精细调节；
   - 每个频段右侧会显示当前增益（dB）。
5. **开始/停止处理**
   - 点击“开始音频处理”按钮，应用将通过选定的输入设备采集声音，经处理后输出到目标设备；
   - 再次点击按钮可停止处理。
6. **观察波形**
   - “实时波形”卡片中同时展示输入与输出波形，便于观察设置对信号的影响。

应用会自动记住以上所有设置，下次启动时自动恢复。

## 安全与注意事项

- **音量安全**：
  - 高放大倍数和高输出音量可能造成听力损伤，请谨慎调节，逐步增加。
  - 长时间使用时建议定期休息，遵循听力健康建议。
- **设备选择**：
  - 在嘈杂环境或外出场景，使用 **入耳式/罩耳式耳机** 通常效果更好。
- **功耗**：
  - 实时音频 + 前台服务 + 唤醒锁会显著增加电量消耗，长时间使用时建议连接电源。
- **异常情况**：
  - 若出现断断续续、卡顿或啸叫（反馈）等问题，可尝试：
    - 降低放大倍数和音量；
    - 调整设备距离或更换耳机类型；
    - 关闭降噪或切换均衡器预设。
